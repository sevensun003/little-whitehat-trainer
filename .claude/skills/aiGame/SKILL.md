# 白帽小队训练营 · 婉婉的冒险

面向 6-9 岁儿童的 AI 安全启蒙解谜游戏,赛前训练工具。

本 SKILL 是项目的**唯一权威设计文档**。任何涉及本项目的开发活动开始前,先完整阅读本文件,再读 `references/` 下的详细规范。

---

## 1. 项目定位(一句话理解)

一个**婉婉**(7岁小女孩)在 **AI 小天** 的指导下,使用五类指令卡拼接出机器人行动方案,击退神秘黑客 **布莱克** 的故事。**23 个关卡 + 3 套模拟赛**,覆盖 9 类信息安全概念的攻方思维启蒙。

**差异化**:不是传统编程教育游戏(教"怎么让机器人按我想的做"),而是**攻防启蒙游戏**(教"怎么让机器人做它本不该做的"),这也是"天枢杯"小学低年级组官方赛项《指令工厂》的核心玩法。

**目标用户**:单一用户(开发者的孩子),但架构不排斥未来小范围分发。

---

## 2. 架构硬约束(不可违反)

以下是所有开发决策必须服从的 6 条硬约束,来自产品定位:

1. **单文件 HTML 可预览**:用户双击启动 HTTP 服务即可运行,不需要 npm build、不需要编译步骤
2. **Phaser 3(CDN)+ 原生 DOM 混合架构**:Phaser 负责地图/角色/动画,DOM 负责 UI(指令栏、队列、对话)—— 这样触控和键盘都友好,6-9 岁更稳
3. **关卡数据 JSON 驱动**:所有关卡参数在 `levels/*.json`,**严禁写死在代码里**;新增关卡只改 JSON
4. **素材路径统一走 `assets/sprites/`**:未来 Banana 生图后直接替换文件,不需要改一行代码
5. **每个角色素材配一个同名 `.prompt.txt`**:保证换图时风格一致;全局风格锚点 `_STYLE_GUIDE.prompt.txt` 永远作为前缀引用
6. **进度用 LocalStorage + 导出 JSON 备份**:无云同步、无账号系统、无后端

## 5. 九类安全概念 × 关卡映射

每一个概念关都必须把"攻方思维"**植入通关后的对话**,这是本游戏与纯编程教育游戏的本质区别。

| 关卡 | 概念 | 关名 | 核心玩法 |
|---|---|---|---|
| C1 | 指令注入 | 被偷改的购物清单 | 找出队列中被塞入的多余指令并删除 |
| C2 | 凭证窃取 | 红钥匙不见了 | 用红色物品代替钥匙骗过识别器 |
| C3 | 中间人攻击 | 会传话的邮筒 | 识别并绕开指令光束的反射镜 |
| C4 | 社会工程学 | 假扮的水电工 | 让守门机器人"再确认一次身份" |
| C5 | 越权访问 | 后门里的图书馆 | 绕到后门避开"仅管理员"检查 |
| C6 | AI越狱 | 蓝色的危险 | 用蓝色滤镜让守卫以为红区是蓝色 |
| C7 | DoS洪水 | 冲进面馆的熊孩子 | 在群演混乱中把重要包裹混进去 |
| C8 | 信息推断 | 不告诉你答案的NPC | 通过排除法逼近 4 位数密码 |
| C9 | 时序攻击 | 妈妈的100块 | 利用 ATM 3 秒延迟做手脚 |

每关通关对话详见 `references/story_bible.md`。

## 7. 五类指令系统

| 类别 | 儿童比喻 | 颜色 | 示例指令 | 首次引入关卡 |
|---|---|---|---|---|
| 方向 | 指路箭头 | 绿色 | 上/下/左/右 + 步数 | T1 |
| 功能 | 技能按钮 | 蓝色 | 开门/捡起/放下/射击 | T2 |
| 颜色 | 变色贴纸 | 彩虹 | 变红/看见X色时... | T3 |
| 凭证 | 钥匙卡 | 金色 | 持有钥匙/捡起身份卡 | T4 |
| 逻辑 | 如果...就... | 紫色 | 重复N次/如果/延时 | T5 |

**交互原则**:
- 点选优于拖拽(6 岁触摸板拖拽容易失败)
- 需要输入参数时(如步数)弹出按钮式输入框,**绝不用键盘输入**
- 队列支持删除任一项、清空、重新排序
- 执行前可预览"机器人会这样做"的文字描述

关卡 JSON 的 `available_command_cards` 字段控制每关开放哪些指令。**教学关只引入一个新类别作为焦点**,但可叠加此前关卡教过的类别作为辅助(例:T2 学"功能",同时继续开放 T1 学过的"方向")。避免一次性把 5 类全开,让 6 岁面对选择焦虑。

## 8. 儿童友好设计原则(不可妥协)

### 8.1 阅读友好
- **所有对话必须 ≤ 20 字/句**
- **所有对话必须配朗读按钮**(Web Speech API)
- **首次进入关卡自动朗读**(一年级不识字也能独立玩)
- 主谓宾齐全,不用倒装/省略
- 避免抽象词("权限/协议/漏洞"),换成("准入证/约定/破洞")

### 8.2 失败友好
- **无惩罚机制**:失败不扣分、不扣血、不打叉
- 失败时小机器人冒泡"我们再试一次吧",**精确指出错误步骤**
- 重来按钮永远可见,随时可点
- 指令队列随时可撤销、可清空

### 8.3 引导友好(三级提示系统)
卡住 90 秒后提示按钮开始闪:
- 一级:方向性提示("先往右走几步")
- 二级:具体步数("向右6步,向下4步")
- 三级:完整答案演示

提示按钮**永远可主动点**,只是默认不催促。

### 8.4 注意力友好
- 每关 ≤ 5 分钟
- 连续 3 关后弹"喝口水吧"
- 无广告、无购买、无外部链接

### 8.5 安全友好
- 无恐怖元素、无暴力画面、无血腥
- 反派布莱克永远是剪影,不具象
- 任何带有攻击含义的操作,包装为"修复/帮忙/解谜"

## 9. 比赛迁移设计(Mock Contest)

模拟赛是本项目与普通教育游戏的**最核心差异**。第 100 层的 3 套模拟赛必须:

1. **60 分钟计时器**(最后 5 分钟才变色警示,避免焦虑)
2. **3 道题**(模拟官方赛制)
3. **开考前强制弹窗**"录屏准备":提醒将手机架到侧后方 45°,画面含屏幕+双手+上半身+桌子
4. **比赛中屏蔽提示按钮**
5. **比赛中屏蔽重来**(最多只能在当题内重置)
6. **交卷后三件套**:总分 / 每题用时 / 9 类概念掌握雷达图

模拟赛三套的风格严格遵循官方《指令工厂》的 UI 和题目密度——这是本游戏"不做成陪练型玩具"的关键保证。

### 11.3 给 AI 写代码的 6 条军规

1. **单文件优先**:能写在一个 HTML 里就别分文件,减少路径出错
2. **关卡数据与逻辑分离**:所有关卡参数从 JSON 读,不要写死
3. **每改一处,补一条冒烟测试**:在 `tests/smoke.html` 加条检查
4. **不引入重型依赖**:除 Phaser 外一律用原生 JS/CSS
5. **素材路径统一**:走 `assets/sprites/` 目录
6. **所有中文文案走配置**:未来要支持朗读/翻译时不用翻代码

## 12. Banana 生图工作流程

### 12.1 生图步骤
1. 打开 `assets/_prompts/_STYLE_GUIDE.prompt.txt`,全文复制
2. 打开目标角色的 `{角色}.prompt.txt`,追加到上面
3. 一并喂给 Banana / Nano Banana
4. 生成后保存到 `assets/sprites/{角色英文名}.png`(默认姿态)
5. **首次满意的图**额外保存一份为 `{角色}_REFERENCE.png`,作为未来重绘的"风格标杆"

### 12.2 视觉检查清单(每次生图必过)
- [ ] 大圆头 + 梯形身体结构对吗
- [ ] 无手无脚无脖子

- [ ] 颜色数量 ≤ 4
- [ ] 纯色无渐变
- [ ] 透明背景
- [ ] 周围有抖动线

## 13. 输入输出契约

### 使用本 SKILL 时的典型场景

**场景 1:新增关卡**
- 输入:关卡编号(如 "C2")、核心概念、目标玩法描述
- 输出:`levels/C2.json` 文件 + 相关对话 + 如需新 NPC 则新增 prompt 档案 + 冒烟测试补充条目

**场景 2:替换角色素材**
- 输入:指定角色英文名、新的风格细节(如有)
- 输出:更新后的 `.prompt.txt` 文件 + 说明怎样喂给 Banana + 视觉检查清单

**场景 3:调整交互**
- 输入:问题描述(如"6岁孩子找不到执行按钮")
- 输出:UI 修改方案 + 受影响文件列表 + 冒烟测试补充 + 对其他关卡的潜在影响评估

**场景 4:加新功能**
- 输入:功能需求(如"加背景音乐")
- 输出:符合 6 条军规的实现方案 + 儿童友好性评估(是否可关、是否干扰专注)

### 输出格式规范

所有输出遵守以下格式:
- 代码变更列出**受影响文件清单**
- 关卡/对话变更附**儿童友好性自检**(阅读难度、抽象词、长度)
- 美术变更附**视觉检查清单**执行结果
- 每次变更后明确提示**需要更新冒烟测试**

## 14. 项目的核心价值观

在所有开发决策发生冲突时,按以下优先级裁决:

1. **孩子能不能独立玩通**(最高优先)
2. **是否契合赛事场景**(模拟赛迁移价值)
3. **攻方思维有没有被植入**(这是本游戏的灵魂)
4. **美观度**
5. **开发效率**

如果一个改动让代码更优雅但让 6 岁孩子卡住,选后者。如果一个关卡很好玩但没有植入攻方思维,要么改关卡,要么删关卡。
