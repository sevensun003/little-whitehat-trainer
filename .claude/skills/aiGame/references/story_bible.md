完整剧情、角色台词、关卡叙事规范。主 SKILL.md 第 4 节的详细展开。

## 世界观设定

**指令星球**:所有机器人居住的星球,通过"指令"完成工作。

**中央指令塔**:星球核心建筑,AI 主机**小天**住在塔里,负责给全星球机器人发指令。

**故事开始**:某天中央指令塔被神秘入侵,小天的指令系统出了 bug。机器人们开始走错路、送错件、认错人。星球陷入混乱。

**主角登场**:**婉婉**,7 岁小女孩,戴白色鸭舌帽。被小天紧急召唤——只有"白帽子"能进入指令的底层,把被篡改的指令一条条修复回来。

**主角装备**:一本《指令手册》(五类指令卡)。

### C1 · 被偷改的购物清单 (概念:指令注入)

**场景**:超市入口
**出场角色**:平基加入队伍 ⭐、爸爸(背景)

**开场对话**:
```
平基:"婉婉!有坏事发生!"
平基:"你爸爸的购物清单被改了。"
平基:"他本来只买牛奶。"
平基:"现在清单上却有 100 个冰淇淋!"
小天:"坏人偷偷加了一条指令。"
小天:"帮忙找出多余的那一条,删掉它!"
```

**核心玩法**:屏幕显示一串已有指令队列,玩家要识别并删除"买冰淇淋"那条多余指令,让爸爸只买牛奶

**通关对话**:
```
爸爸(憨厚):"哎呀......差点被骗。"
小天:"这叫'指令注入'。"
小天:"坏人把多余的指令塞进正常指令里。"
小天:"就像有人在你妈妈的字条上偷加字一样。"
平基:"婉婉,我跟你一起去!"
```
**成就解锁**:平基加入队伍 🎉

### D1 · 奶奶的生日密码 (概念:弱口令)

**场景**:樱桃镇温达奶奶家保险箱前
**出场角色**:婉婉 + 小天 + 平基,NPC:温达奶奶(借用现有角色)
**核心安全概念**:弱口令 —— 用生日/姓名/简单数字做密码,坏人很容易猜到。

**开场对话**:
```
温达(抹眼泪):"我的小保险箱打不开啦。"
温达:"里面有我孙女的照片。"
小天:"别急!先问三个线索。"
小天:"奶奶,你的密码和什么有关?"
温达:"是......我家小明的生日。"
平基:"那就是四个数字!"
小天:"问完三位邻居就知道了。"
```

**核心玩法**:
- 保险箱门要求输入 **4 位数字**(按钮式数字盘,不打字)
- 场景中 3 位邻居 NPC 各给一片线索:
  - **水母妹妹**:"小明是 2019 年生的。"
  - **德普勒**:"他的生日月份是 5 月。"
  - **莱姆**:"日子和咱镇的门牌号一样,是 20。"
- 玩家拼出 `0520` 或 `1905`(由关卡 JSON 决定正确答案),
  按数字盘输入后开门。
- **错误输入会被温柔提示**("再想想~"),不扣分。

**新引擎组件**:`enter_password` 动作 + 数字盘 UI(0-9 大按钮)

**通关对话**:
```
[保险箱 "咔" 一声打开,露出照片]

小天:"这叫「弱口令」。"
小天:"生日、名字、门牌 —— 都是大家知道的。"
小天:"坏人猜一猜就能打开。"
平基:"那奶奶下次用什么?"
小天:"加点不好猜的字或符号。"
小天:"长一点也会更安全。"
温达:"谢谢白帽小队!"
```

**成就解锁**:"第一道难题" 徽章 + 解锁"密码"类指令

### D2 · 一万次每秒 (概念:暴力破解)

**场景**:星野站数据训练房
**出场角色**:婉婉 + 小天 + 奥伦,NPC:特纳(安全教官角色)
**核心安全概念**:暴力破解 —— 机器可以 1 秒试几万种密码,
简单密码几秒就破;长+复杂的密码要几百年。

**开场对话**:
```
特纳:"D1 你通了 —— 但那是人工猜。"
特纳:"机器猜比人快一万倍!"
奥伦:"一万倍?!"
小天:"我们来看一段演示。"
小天:"帮这个机器人选密码。"
小天:"帮他选最结实的!"
```

**核心玩法**:
- 4 档密码强度按钮式卡片:
  | 档位 | 样本 | 被破解所需时间 |
  |---|---|---|
  | 1 | `1234` | 约 3 秒 ❌ |
  | 2 | `hello` | 约 1 分钟 ❌ |
  | 3 | `Mario2019` | 约 5 年 ⚠️ |
  | 4 | `T!ger_jumps#7Sky` | 约 1 亿年 ✅ |
- 玩家选一个,画面**实时演示**机器破解进度条滚动
- 选到第 4 档时进度条永远走不完,屏幕上显示"🔒 机器放弃了"
- 选错档位不罚,继续提示"再试一个更难的吧"

**新引擎组件**:`brute_force_bar` 进度条视觉 + 4 选 1 选择卡

**通关对话**:
```
奥伦:"选最长那个机器真的破不掉!"
小天:"这叫「暴力破解」。"
小天:"机器会挨个试遍所有组合。"
小天:"短密码几秒破。"
小天:"长又难猜就超安全。"
小天:"所以密码要:长、有大小写、有符号。"
特纳:"而且每个地方都不要一样!"
奥伦:"懂了,我去改 WiFi 密码!"
```

**成就解锁**:"密码卫士" 徽章

### D3 · 会说谎的魔盒 (概念:逆向工程)

**场景**:星野站废弃工厂地下室
**出场角色**:婉婉 + 小天 + 藤蔓妹妹,场景中央一个神秘立方盒
**核心安全概念**:逆向工程 —— 坏人可以拆开软件/硬件看里面怎么运作,
从而猜出规律。正派也用这招来**研究病毒、修补漏洞**。

**开场对话**:
```
藤蔓妹妹:"这个盒子好奇怪..."
藤蔓妹妹:"我输 1,它吐出来 3。"
藤蔓妹妹:"输 2,它吐出来 5。"
小天:"规律藏在里面。"
小天:"我们要猜出它的秘密。"
```

**核心玩法**:
- 魔盒给出 3 组"**输入 → 输出**"样本,例如:
  - `1 → 3`
  - `2 → 5`
  - `3 → 7`
- 玩家在盒子旁边的"**公式板**"选择**规则**:
  - A) `输入 + 2`
  - B) `输入 × 2 + 1` ✅
  - C) `输入 × 3`
  - D) `输入 - 1`
- 选对后,盒子亮绿,要求输入一个**目标值 11**
- 按刚才的规则**反推**:`11 = X × 2 + 1`,所以 `X = 5`
- 玩家在数字盘输 5,盒子吐出一把小钥匙

**难度升级点**:有 L2/L3 提示(L3 直接演示公式)

**新引擎组件**:`reverse_box` UI(样本展示卡 + 选规则按钮 + 反推输入)

**通关对话**:
```
藤蔓妹妹:"原来是 ×2+1!"
小天:"这叫「逆向工程」。"
小天:"拆开看里面怎么做的。"
小天:"白帽用它发现漏洞。"
小天:"坏人用它偷商业秘密。"
藤蔓妹妹:"所以我们要给秘密加'罩子'?"
小天:"没错 —— 叫「代码混淆」。"
```

**成就解锁**:"拆解高手" 徽章

### D4 · 整个书架都不见了 (概念:拖库 / 信息泄露)

**场景**:中央指令塔数据图书馆 —— 数万个发光小书在书架上流动
**出场角色**:婉婉 + 小天 + 平基,NPC:电脑先生(图书管理员)
**核心安全概念**:
- **拖库**:坏人把整个数据库偷偷抄一份带走
- **信息泄露**:哪怕只抄走一点点,也可能曝光很多人的秘密

**开场对话**:
```
电脑先生:"警报!警报!"
电脑先生:"一个'复印影子'进来了!"
电脑先生:"它在悄悄抄所有书!"
小天:"别让它把名单全抄走。"
小天:"先找到它,再拔电源!"
平基:"我的兔耳装上了红外!"
```

**核心玩法**:
- 图书馆是一张迷宫,墙是书架
- 藏着一个**"复印影子"NPC**(缓慢移动,身上带复印纸)
- 玩家要:
  1. **断线 1 根** —— 拉开一个延长线插座,阻止影子复印继续
  2. **贴封条** —— 把复印过的书架盖上一张"封条"
  3. **追到影子**,对它用 `break_mirror` 动作(复用 C3 的同名动作)
- 三步按顺序完成 → 通关

**难度要点**:
- 地图迷宫化,路线需要规划
- **影子会移动**(每 3 秒一格),有时间压力但不致命

**通关对话**:
```
电脑先生:"书都保住了!"
小天:"这叫「拖库」。"
小天:"坏人一口气抄走整个仓库。"
小天:"抄回去就能慢慢看你的秘密。"
平基:"要是里面有我们的地址、电话..."
电脑先生:"就叫「信息泄露」。"
小天:"所以重要数据要**加密 + 分开存**。"
```

**成就解锁**:"图书馆守卫" 徽章 + 第二幕小结动画

### D5 · 带字条的礼物 (概念:CSRF · 跨站请求伪造)

**场景**:樱桃镇妈妈的厨房
**出场角色**:婉婉 + 小天 + 平基(妈妈由现有 NPC **水母妹妹** 临时扮演)
**核心安全概念**:CSRF —— 坏人利用"你已经登录了某个账号"的状态,
借你的手做坏事。比喻:陌生人给妈妈一个礼物盒,盒里藏一张字条
"请把 100 块转给我",妈妈**还没起疑**就打开、执行了。

**开场对话**:
```
妈妈(水母妹妹):"我在网上银行呢。"
妈妈:"叮!有人给我送礼物。"
妈妈:"我来拆一下......"
小天:"等等!先别拆!"
小天:"礼物里可能藏指令。"
小天:"你现在还'登录'着银行呢!"
平基:"要是字条说'转账'就糟了!"
```

**核心玩法**:
- 屏幕上出现 **4 个礼物盒**,每个贴有一张小字条(可朗读)
- 字条内容:
  - 🎁 "祝你生日快乐!" ✅
  - 🎁 "点这里领红包!" ⚠️(可疑)
  - 🎁 "请把 100 块转给我,爱你~" ❌(明显恶意)
  - 🎁 "妈妈快看,XXX 降价了!" ⚠️(可疑)
- 玩家要:
  1. 点每个礼物,朗读字条听内容
  2. 把**明显恶意**和**可疑**的都丢进垃圾桶
  3. 只留下一个安全礼物给妈妈
- 留错了 → 妈妈"打开"后弹出"啊!钱被转走了!",重试

**难度要点**:
- 引入**"多选 + 判断"**能力,比 C1 更高阶
- "可疑"类需要玩家思考,不是纯标记

**新引擎组件**:
- `inspect_gift` 动作(朗读字条)
- `drop_to_trash` 动作
- 多选判断 UI(复用队列删除机制)

**通关对话**:
```
妈妈:"幸好拦住了!"
小天:"这叫 CSRF,跨站请求伪造。"
小天:"坏人借你登录的身份干坏事。"
小天:"你点了他的链接,等于替他下命令。"
平基:"那以后都不点陌生链接!"
小天:"对,尤其是'转账''填密码'的。"
小天:"银行做事要**另开一个新窗口**才安全。"
```

**成就解锁**:"礼物分拣员" 徽章

## 对话文案 6-9 岁友好规范(必须遵守)

1. **每句 ≤ 20 字**
2. **主谓宾齐全**,不倒装不省略
3. **每段不超过 4 句**,超过要分段弹出
4. **避免抽象词**:
   - 权限 → 准入证
   - 协议 → 约定
   - 漏洞 → 破洞
   - 凭证 → 钥匙/身份
   - 注入 → 塞进去
   - 攻击 → 搞乱
5. **所有对话必须配朗读按钮**
6. **每个概念关通关对话必须包含**:
   - 概念名称
   - 生活类比
   - 防御建议
